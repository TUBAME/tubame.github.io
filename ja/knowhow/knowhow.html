1.はじめに


1.1.本ドキュメントの紹介


本マニュアルは、 ナレッジ管理ツール を使用する為に、インストールから利用するまでの手順について簡易に説明したマニュアルです。



1.2.本ツールの紹介


本ツールは Eclipse プラグインとして動作し、ナレッジを GUI 上で容易に作成、管理することができるツールです。 ナレッジには、移植箇所を抽出するための検索条件、移植難易度、移植工数 (移植概算ライン) 等が設定可能です。 登録されるナレッジは XML 形式で管理されます。 大きく以下の情報を扱うことが可能です。

Table 1. 本ツールで扱う情報

項目説明
ナレッジのドキュメント情報ドキュメント情報を HTML 形式で保存できます。保存したファイルを ナレッジベース検索ツール にコピーすれば、検索結果からリンク参照することができます。
移植箇所の検索情報検索情報を XML 形式で保存できます。保存したファイルを ナレッジベース検索ツール の入力として使用すれば、検索ロジックとして実行することができます。

3.インストール


3.1.インストール


事前に 動作環境 に記載されている環境等の準備を整えてください。 準備が整っていることを確認し、以下の手順を実施します。

  1. ダウンロード

    プラグインをダウンロードします

    Note

    https://github.com/TUBAME/migration-tool/releases

  2. 解凍

    ダウンロードしたファイルを解凍します。 解凍すると tubame.knowhow_* という名前のディレクトリが作成されます。

    Note

    * 部分はプラグインのバージョンによって変わります。

    Important

    Windows 上で zip を解凍する場合、パス長の制限により解凍に失敗する場合があります。 失敗した場合は、 C:\temp のような短いパスに解凍してください。

  3. Eclipse へインストール

    解凍したディレクトリを、Eclipse インストールディレクトリ下の plugins ディレクトリに移動またはコピーします。

  4. インストール確認

    以下のように Eclipse メニューを選択します。

    Help → About Eclipse → Installation Details → Plug-ins

    Plug-in Id の部分に、以下のプラグイン名 tubame.knowhow が存在することを確認してください。

  5. インストール完了

    これでインストール完了です



4.利用方法


4.1.利用の流れ


ツールの利用の流れを解説します。



4.1.1.Eclipseプロジェクトの作成


以下の手順を実施します。

  1. Eclipse を起動

  2. プロジェクト作成ウィザードの起動

    File → New → Project...

  3. Project の作成

    Project → Next

    New Project ウィザードで、移植元の環境に応じて適宜設定を行いプロジェクトを作成します。



4.1.2.ツールの起動


以下の手順を実施します。

  1. パースペクティブの起動

    Window → Open Perspecive → Other → ナレッジ管理ツール

  2. ツールの初期化

    パースペクティブを起動すると自動的にツールの初期化処理が開始されます。初期化が完了するまでお待ちください。

    Note

    XML のマーシャル / アンマーシャルクラスの初期化に時間を要する場合があります。

    Note

    初期化処理はキャンセルできません。 Cancel ボタンをクリックしても無視されます。



4.1.3.ナレッジの作成


ナレッジの作成方法を以下に記載します。

  1. ナレッジ XML ファイルの新規作成

    File → New → Other → ナレッジ XML ファイル

    Note

    作成済みの ナレッジ XML ファイルを開く場合は。該当ファイル上で右クリックして以下のようにメニューを選択してください。

    Open With → Other → ナレッジエディタ

  2. ナレッジエントリービューの操作

    個々のナレッジは ナレッジエントリービュー 上で追加・参照・更新・削除します。 作成したナレッジはアイテムとしてドキュメント作成等に活用できます。 ナレッジエントリービュー は、以下のいずれかの方法で起動します。

    • ナレッジ XML 新規作成後に自動起動

    • Eclipse のビュー表示メニューからの起動

      Window → Show View → Other → ナレッジ管理ツール → ナレッジエントリービュー

  3. ナレッジを構成するアイテム群の作成

    アイテムの作成は以下の手順で実施します。最初はアイテムが 1 つもないため、必ず カテゴリ を作成することになります。

    (アイテムの選択) → 右クリック → 任意の登録メニュー → 登録Wizard起動

    Note

    アイテムを未選択の場合は カテゴリ のみ作成可能です。

    Note

    上記の他に、コピー&ペーストによる作成も可能です。

    ナレッジを構成するアイテムは、以下の種類があります。 以下に各アイテムの説明を記載します。

    Table 1. アイテムの種類

    項目説明
    カテゴリドキュメント構成の章・節にタイトルに相当すアイテムです。タイトル情報のみ保持します。
    ナレッジ ドキュメント構成の章・節に記述されるナレッジコンテンツに相当するアイテムです。 DocBook#Artile 形式のコンテンツ情報のみ保持します。
    チェックアイテム 該当ナレッジの移植判断情報を保持するアイテムです。 検索手順 、 移植要因 、 難易度 などの情報を保持します。
    検索情報 ナレッジベース検索ツール が検索の際に使用する検索ロジックとなるアイテムです。


  4. ナレッジエディタの操作

    登録されたアイテムの編集や、ドキュメント構成の作成を行います。 ナレッジエディタ には以下の種類のタブがあります。 以下に各タブの説明を記載します。

    Table 2. タブの種類

    項目説明
    ドキュメンテーション 作成する HTML ドキュメントの章構成を編集することができます。 見出しを作成後、 ナレッジエントリービュー から カテゴリ をドラッグ&ドロップすることでドキュメントを構成していきます。
    ナレッジ詳細 ナレッジのコンテンツを DocBook#Article 形式で編集するエディタです。エディタ上で右クリックを行い、タグのテンプレートを挿入することもできます。
    Important

    独自にセクションが作成され章構成が崩れるのを防ぐため、 <section> タグは使用できません。

    チェックアイテム ナレッジエントリービュー で選択している チェックアイテム情報 および 検索情報 を編集することができます。


    以下に チェックアイテム タブで編集する チェックアイテム情報 および 検索情報 の各入力項目の説明をします。

    Table 3. チェックアイテム情報

    項目説明
    検索手順登録するナレッジとしてどのような移植箇所をどのように検索するのかの概要を文章で入力します。
    検索実施 ナレッジベース検索ツール で検索する際に、実際に検索対象とする場合はチェックを有効にします。
    移植要因移植が必要となる理由として該当する項目を選択します。
    難易度 移植作業の難易度を選択します。
    Note

    選択基準は以下となります。

    • 高

      実装内容の大規模な修正が必要で、実装や設定等の大幅な見直しが必要

    • 中

      実装内容を考慮した小規模な修正が必要

    • 低

      実装内容を考慮せず軽微な修正で対応可能

    • 不明

      難易度不明

    • 移植不可

      移植不可
    難易度詳細移植作業の詳細な難易度を選択します。
    Note

    選択基準は以下となります。

    • 高

      難易度が 高 であるもの

    • 中

      難易度が 中 であるもの

    • 低1

      難易度が 低 であり、置換レベルで対応不可能であるもの

    • 低2

      難易度が 低 であり、置換レベルで対応可能であるもの

    • 不明1

      難易度が 不明 であり、JBossにて該当機能がなく難易度を判断できないもの

    • 不明2

      難易度が 不明 であり、JBossに該当機能があるかどうかも不明で難易度を判断できないもの

    • 移植不可

      難易度が 移植不可 であるもの
    目視確認設定したロジカルな検索条件だけでは移植項目が特定できない場合の目視による調査方法または、判定基準等を入力します。
    ヒアリング確認開発元に確認しなければ移植の必要性が判断できない項目の場合の開発元への確認内容を入力します。


    Table 4. 検索情報

    項目説明
    検索対象ファイル検索対象とするファイル名を入力します。正規表現での入力が可能です。
    検索キーワード 1検索モジュールに渡すパラメータを入力します。正規表現や XPath での入力も可能です。
    Note

    入力するパラメータについては、検索モジュールによって変わります。検索モジュール概要を参考にパラメータを入力してください。
    検索キーワード 2検索モジュールに渡すパラメータを入力します。正規表現や XPath での入力も可能です。
    Note

    入力するパラメータについては、検索モジュールによって変わります。検索モジュール概要を参考にパラメータを入力してください。
    検索モジュール検索モジュール概要を参考に検索したいモジュールを選択します。
    Note

    未選択の場合は、標準検索機能により検索が実行されます。標準検索機能は、検索対象ファイルでヒットしたファイルに対して 検索キーワード 1 で検索後、 検索キーワード 2 でさらに検索を行います。
    検索モジュール概要検索モジュールの概要説明が表示されます。
    ライン数算出検索にヒットした場合に ライン数 に入力した値を移植工数として加算する場合はチェックを有効にします。
    非算出理由 ライン数算出 を無効にする場合は、その理由を入力します。
    不明 / TODO ライン数 が不確定な場合に、該当する項目を選択します。
    ライン数移植に要するおおよその修正ライン数を入力します。
    ライン数根拠入力した ライン数 の根拠を入力します。
    調査内容修正ライン数を算出するための調査方法等を入力します。




4.1.4.ドキュメントの確認


作成したナレッジドキュメントを Eclipse 標準ブラウザで確認することができます。 以下の 2 種類の確認方法があります。

  • ドキュメント全体での確認

    ツールバー上の HTML プレビュー ボタンで、 ドキュメンテーション タブにて構成したドキュメント全体を確認できます。

  • アイテム単体での確認

    ナレッジ詳細 タブにて作成したアイテム単体でのドキュメントを確認できます。

    ナレッジ詳細 → 右クリック → HTML プレビュー



4.1.5.ファイルの保存


作成したナレッジをファイルとして保存できます。 以下の 2 種類の確認方法があります。

  • XML 保存

    ツールバー上の XML 保存 ボタンで、 XML 形式でファイルを保存することができます。

    Note

    保存したファイルは、 ナレッジ管理ツール で再度編集することができます。

    Note

    保存したファイルは、 ナレッジベース検索ツール の入力ファイルとして利用できます。

  • HTML 保存

    ツールバー上の HTML 保存 ボタンで、 HTML 形式でファイルを保存することができます。

    Important

    保存したファイルは、 ナレッジ管理ツール で再度エディットすることはできません。

    Note

    保存したファイルは、 ナレッジベース検索ツール のリンク先ファイルとして利用できます。



5.アンイストール


5.1.アンインストール


インストールした tubame.knowhow_* を削除してください。以上で、アンインストールは完了です。

Note

* 部分はプラグインのバージョンによって変わります。

Note

削除した後にEclipseを起動する際は -clean のオプションを付与して起動してください。
